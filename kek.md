Модуль: описание проекта, бота 
1. Введение (для кого этот документ: пользователи и проверяющие) 

2. начало работы
2.1 После нажатия на /start, вы должны ввести свои данные от сайта ЧЮК (пароль не сохраняется в сообщениях, и к тому же админ видит только логин, без пароля)
2.2 После ввода нужных данных можно например получить расписание своей группы, либо посмотреть статистику по неделе, расписание преподавателя (затем его можно добавить в избранное)
2.3 Безопасность в боте: в боте используется современное шифрование aes 256 и частично gostcrypto для важных данных (пароль и тд)

3. Функциональные возможности 
3.1 Примеры полезных и основных функций: Статистика расписания (самый загружнный день, сколько часов в неделе, среднее кол-во пар в день), получение расписания (благодаря парсеру для html, затем после получения в сообщении им можно поделиться, нажав кнопку "поделиться", проверка статуса бота и сайта (работает ли, проходит авторизация))
3.2 Имеется админ-панель (просмотр пользователей, общая статистика, бан/разбан, система поддержки, авторассылка, глобальное уведомление для всех пользователей, возможность поиска пользователям по id или по логину)
3.3 Есть функция "избранные преподаватели", где при нажатии на эту кнопку открывается меню избранных преподавателей, которых пользователь добавляет сам после выдачи расписания препода, либо просто через добавление (значок плюса) в том же меню
3.4 Есть функция "зачетная книжка", при нажатии на эту кнопку выдается в сообщении зачетная книжка ЧЮК определенного пользователя из определенной группы (определяется по введенным пользователем данным), и затем пользователь может увидеть свою зачетную книжку (сданные предметы и тд)

4. Помощь пользователям
4.1 Если что то не работает (выдача расписания, еще что то, багается что либо), то можно отправить тикет (запрос) в поддержку с полным описанием проблемы, которая случилась у вас, и админы вам помогут решить этот вопрос, ответив на репорт
4.2 Имеется проверка бота и проверка сайта (т.е лежит ли сайт, что-то в боте не так, выполняется проверка через тестовые модули авторизации и прогонки бота по некоторым функциям)
4.3 Доступна авторассылка для пользователя, чтобы обеспечить легкость использования бота (можно самому выбрать нужное время и день недели для авторассылки расписания, например "суббота 15 30") и в субботу в назначенное время пользователю вышлется расписание на неделю 

5. Технические детали
5.1 Имеется файлы (.sh, .ps1) для запуска бота на сервере в один клик (т.е запускаешь нужный файл, и бот активируется на удаленном сервере: устанавливаются зависимости, запускается нужный файл и тд)

6. Итоговый пример использования 
Пользователь вводит в бота свои данные (логин и пароль), бот автоматически пробует авторизироваться через сайт ЧЮКа, и если какие либо данные не совпадают, то он соответственно выведет (неверный логин или пароль)
Если все верно, то появляется меню кнопок для работы пользователя с ботом и активируется возможность использования всеми функциями бота для обычного пользователя, и админскими функциями для админов
